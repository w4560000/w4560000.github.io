    GCP Cloud Build 範例

    Devops > GCP GCP
    發布時間 : 2022-04-02 22:19
    字數:274 觀看數 :
    評論:
    1. 本文以編譯專案 Dockerfile 至 GCR 當範例
      1. 在專案上 加上 cloudbuild.yaml
      2. 設定Cloud Build 觸發條件
      3. 設定好後 在專案遠端分支異動版本

    Cloud Build 是在 GCP 上的一個提供 Build, test, deploy 的一個 serverless CI/CD 平台
    相關文件: https://cloud.google.com/build
    相關費用: https://cloud.google.com/build/pricing?hl=zh-TW

    本文以編譯專案 Dockerfile 至 GCR 當範例

    在專案上 加上 cloudbuild.yaml

    cloudbuild.yaml 參數文件 可參考: https://cloud.google.com/build/docs/build-config-file-schema

    steps:
  # 編譯映像檔
  - name: 'gcr.io/cloud-builders/docker'
    id: bx-test-project
    dir: 'Test_Project/' 
    env: ['NODE_ENV=$_NODE_ENV']
    args:
      - 'build'
      - '-f'
      - 'Dockerfile'
      - '--build-arg'
      - 'NODE_ENV=$_NODE_ENV'
      - '-t'
      - 'gcr.io/$PROJECT_ID/bx-test-project:ci-$_NODE_ENV-$SHORT_SHA'
      - '.'

  # 上傳映像檔
  - name: 'gcr.io/cloud-builders/docker'
    id: api-push
    args:
      - 'push'
      - 'gcr.io/$PROJECT_ID/bx-test-project:ci-$_NODE_ENV-$SHORT_SHA'

    新增完後 上傳至 Cloud Source Repositories

    設定Cloud Build 觸發條件

    1. 設定觸發方式 & Cloud Source Repositories 專案來源

    2. 設定 Cloud Build 設定檔

    　需注意cloudbuild.yaml檔位置, 若不再跟目錄 則 設定觸發條件 cloudbuild.yaml路徑要調整

    設定好後 在專案遠端分支異動版本

    • 待 Cloud Build trigger成功後, 即可至 Container Registry 查看 Docker Image
